PENDAHULUAN
Latar Belakang & Tujuan Proyek

Machine Learning merupakan cabang dari kecerdasan buatan yang memungkinkan sistem belajar dari data tanpa diprogram secara eksplisit. Dalam konteks kesehatan, machine learning dapat dimanfaatkan untuk membantu proses diagnosis penyakit secara lebih cepat dan akurat, sehingga berpotensi menyelamatkan nyawa melalui deteksi dini.

Pada proyek ini, dikembangkan sistem klasifikasi untuk mendeteksi kanker payudara menggunakan dataset Breast Cancer Wisconsin (Diagnostic) dari UCI Machine Learning Repository. Dataset ini banyak digunakan dalam literatur ilmiah dan merupakan benchmark standar untuk algoritma klasifikasi dalam domain medis.

Relevansi Proyek: Kanker payudara merupakan kanker paling umum pada wanita secara global. Sistem klasifikasi berbasis ML berpotensi menjadi alat bantu diagnosis yang cepat, objektif, dan konsisten bagi tenaga medis.

Tujuan utama proyek ini adalah:

1
Memahami konsep dasar machine learning dan problem framing untuk kasus klasifikasi biner pada data medis.
2
Melakukan audit data dan preprocessing yang sistematis (penanganan missing value, encoding, scaling).
3
Mengimplementasikan dan membandingkan tiga algoritma klasifikasi: KNN, Decision Tree, dan Naive Bayes.
4
Mengevaluasi performa model menggunakan confusion matrix, accuracy, precision, recall, dan F1-score.
5
Merekomendasikan model terbaik berdasarkan performa, kompleksitas, dan interpretabilitas.

Proyek ini berkontribusi langsung pada pencapaian Sub-CPMK A1164405-81.1, Sub-CPMK A1164405-101.1, dan Sub-CPMK A1164405-81.2 sesuai RPS Pembelajaran Mesin. Seluruh proses menggunakan random_state=42 untuk reproduksibilitas penuh.

PROBLEM FRAMING
Soal 01 — Konsep Pembelajaran Mesin & Formulasi Masalah

Pengertian Pembelajaran Mesin: Machine Learning adalah bidang ilmu yang mempelajari algoritma dan model statistik yang memungkinkan sistem komputer melakukan tugas tertentu tanpa instruksi eksplisit, melainkan dengan mengandalkan pola yang dipelajari dari data. Terdapat tiga paradigma utama:

✅ DIGUNAKAN
Supervised Learning

Belajar dari data berlabel untuk memprediksi output (klasifikasi/regresi). Cocok karena label diagnosis (B/M) tersedia.

Tidak Digunakan
Unsupervised Learning

Menemukan pola tanpa label (clustering, reduksi dimensi). Tidak sesuai karena label sudah tersedia.

Tidak Digunakan
Semi-supervised

Campuran berlabel & tidak berlabel. Tidak relevan karena seluruh data memiliki label diagnosis.

Alasan Ilmiah Kasus Ini Termasuk Klasifikasi: Masalah ini termasuk supervised binary classification karena: (1) target bersifat kategorikal biner (B/M atau 0/1), bukan nilai kontinu; (2) tujuan memprediksi label kelas keganasan tumor, bukan mengelompokkan data tanpa supervisi; (3) tersedia label ground truth dari diagnosis patologi; dan (4) metrik evaluasi menggunakan confusion matrix, accuracy, precision, recall, dan F1-score yang khas untuk klasifikasi.

Framing Masalah: Diberikan vektor fitur x ∈ ℝ30 (30 karakteristik sel tumor), prediksi label y ∈ {0=Benign, 1=Malignant}. Ini adalah masalah klasifikasi biner supervisi dengan implikasi klinis tinggi — false negative (kanker tidak terdeteksi) lebih berbahaya daripada false positive.

Potensi Tantangan Data: (1) Class imbalance ringan (62.7% Benign vs 37.3% Malignant) — tidak kritis sehingga tidak memerlukan oversampling SMOTE; (2) skala fitur sangat beragam (area ~1000, fractal_dimension ~0.05) sehingga scaling wajib untuk KNN dan NB; (3) beberapa fitur berkorelasi tinggi (radius, perimeter, area) yang dapat memengaruhi asumsi independensi Naive Bayes; (4) tidak ada missing value sehingga tidak diperlukan imputasi.

DATA PREPROCESSING
Soal 02 — Pipeline Persiapan Data

Pipeline preprocessing dirancang secara sistematis untuk memastikan kualitas data sebelum pemodelan. Seluruh langkah didokumentasikan dengan justifikasi ilmiah yang kuat.

1
Load & Inspect: Memuat dataset via sklearn.datasets.load_breast_cancer(), memeriksa shape (569×32), tipe data, dan statistik deskriptif.
2
Hapus Kolom ID: Kolom ID unik tidak memiliki nilai prediktif dan dapat menyebabkan overfitting, sehingga dihapus sebelum pemodelan.
3
Encoding Target: Konversi target dari label string (M, B) ke numerik biner: M → 1 (Malignant/ganas), B → 0 (Benign/jinak) menggunakan LabelEncoder.
4
Stratified Train-Test Split: Pembagian data 80:20 dengan stratify=y dan random_state=42 untuk menjaga proporsi kelas di train dan test set.
5
Feature Scaling (StandardScaler): Normalisasi fitur ke zero mean dan unit variance. Krusial untuk KNN (berbasis jarak Euclidean) dan Naive Bayes Gaussian.

Soal 03 — Strategi Fitur & Desain Eksperimen

Seluruh 30 fitur digunakan tanpa seleksi manual karena semua fitur bersifat numerik, relevan secara medis, dan telah melalui proses validasi klinis dalam pembuatan dataset. Untuk analisis eksplorasi, dilakukan visualisasi feature importance menggunakan Random Forest sebagai referensi.

10 Fitur Paling Penting — Referensi Random Forest

Strategi Pembagian Data: Menggunakan train_test_split dengan test_size=0.2 (80:20), stratify=y untuk menjaga proporsi kelas di train dan test, serta random_state=42 untuk reproduksibilitas penuh. Alasan: (1) dataset berukuran sedang (569), 20% test (~114 sampel) cukup untuk evaluasi stabil; (2) stratified mencegah bias kelas; (3) hold-out sederhana sesuai untuk perbandingan model tanpa tuning lanjutan.

Kesesuaian Algoritma terhadap Dataset: KNN cocok karena data memiliki pola lokal yang kuat dan scaling sudah dilakukan; Decision Tree cocok karena interpretabilitas tinggi dan menghasilkan aturan if-then yang dapat dijelaskan kepada dokter; Naive Bayes cocok karena cepat dan memberikan output probabilitas yang intuitif untuk konteks medis.

ANALISIS HASIL, INTERPRETABILITAS & REKOMENDASI
Soal 05 — Model Terbaik & Skenario Penggunaan
🏆 Model Terbaik: K-Nearest Neighbors (KNN k=5)
Direkomendasikan sebagai model utama berdasarkan bukti empiris dari eksperimen.

Alasan Pemilihan KNN sebagai Model Terbaik:

1
Akurasi tertinggi 96.49% dan F1-score 0.9767, jauh unggul dibanding model lain.
2
Recall sangat tinggi 97.67% — sangat penting untuk meminimalkan kanker yang tidak terdeteksi (false negative).
3
Precision tinggi 97.67% — false positive rendah, mencegah kecemasan pasien yang tidak perlu.
4
Non-parametrik — tidak membuat asumsi distribusi data, lebih robust untuk data medis yang kompleks.
5
Mudah diimplementasikan dan dipahami secara konseptual oleh tim medis non-teknis.

Error Analysis: Dari 114 sampel test, KNN hanya salah klasifikasi 4 kasus (1 FP, 3 FN) — error rate hanya 3.51%. Ini jauh lebih baik dibandingkan Decision Tree (8 error) dan Naive Bayes (9 error). 3 false negative pada KNN merupakan kasus borderline yang memang secara karakteristik berada di batas antara kelas Benign dan Malignant.

⚠ Catatan Klinis: Dalam konteks medis, false negative (kanker tidak terdeteksi) jauh lebih berbahaya daripada false positive. Oleh karena itu, Recall menjadi metrik prioritas. KNN memiliki Recall 97.67%, artinya hanya 3 dari 43 kasus kanker yang tidak terdeteksi.

Analisis Probabilitas: Model juga dapat menghasilkan probabilitas prediksi menggunakan predict_proba(). Contoh output untuk sampel baru:
